# renew-ssl-if-needed.sh

Простой и надёжный bash-скрипт для **умного обновления** сертификатов Let's Encrypt только при реальной необходимости.

Проверяет срок действия всех сертификатов → если хотя бы один истекает через **≤ 25 дней** → запускает `certbot renew` → при успешном обновлении перезагружает nginx.

## Зачем именно этот скрипт?

Большинство людей просто ставят `certbot renew` в cron раз в день/12 часов.  
Это работает, но:

- каждый раз запускается certbot (лишняя нагрузка)
- каждый раз certbot что-то пишет в логи
- nginx перезагружается даже тогда, когда ничего не обновилось

Этот скрипт делает всё **интеллектуально**:

- смотрит реальный остаток дней
- renew запускается **только** если нужно
- nginx перезагружается **только** если сертификат(-ы) реально обновились

## Возможности

- Порог срабатывания настраивается (по умолчанию 25 дней)
- Логирование в файл (опционально)
- Проверка, что certbot реально что-то обновил (а не просто запустился)
- Устойчив к ошибкам (set -euo pipefail)
- Работает с несколькими сертификатами одновременно
- Минимум внешних зависимостей

## Установка

```bash
# Сделать исполняемым
chmod +x renew-ssl-if-needed.sh

# Переместить в удобное место (рекомендуется)
sudo mv renew-ssl-if-needed.sh /usr/local/bin/
```
